# ✅ VALIDATION — СТРАТЕГІЧНЕ ПРОЕКТУВАННЯ РЕАЛІЗАЦІЇ ГОЛОВНИХ ОРІЄНТИРІВ

👉 Мета:
швидко перевірити, чи є достатньо якісне стратегічне проектування реалізації для тих головних орієнтирів, які цього потребують

---

## 1. ЧИ ПРАВИЛЬНО ВИБРАНІ ОРІЄНТИРИ ДЛЯ ЦЬОГО РІВНЯ?

- [ ] стратегічне проектування зроблено тільки для орієнтирів зі стратегічною або змішаною моделлю реалізації
- [ ] проектні і підтримуючі орієнтири не були штучно протягнуті через стратегічну машину

👉 якщо ні → рівень невалідний

---

## 2. ЧИ Є ФАКТОРИ РЕАЛІЗАЦІЇ?

Для кожного релевантного головного орієнтиру:

- [ ] визначено фактори реалізації
- [ ] фактори описані не як задачі, а як умови / сили / ризики / невизначеності
- [ ] фактори записані в причинно-наслідковій формі
- [ ] фактори прив’язані до конкретного орієнтиру

👉 якщо ні → рівень невалідний

---

## 3. ЧИ ВІДФІЛЬТРОВАНІ ФАКТОРИ?

- [ ] відкинуті дії, задачі, абстракції без механіки
- [ ] для кожного орієнтиру залишено не більше 5 активних факторів
- [ ] активні фактори реально впливають на реалізацію орієнтиру

👉 якщо ні → рівень невалідний

---

## 4. ЧИ Є СТРАТЕГІЧНІ НАПРЯМКИ РЕАЛІЗАЦІЇ?

Для кожного релевантного головного орієнтиру:

- [ ] сформульовано 2–3 стратегічні напрямки реалізації
- [ ] ці напрямки не є задачами
- [ ] вони прямо відповідають на активні фактори
- [ ] для кожного напрямку зрозуміло, як він впливає на реалізацію орієнтиру

👉 якщо ні → рівень невалідний

---

## 5. ЧИ Є TRADE-OFFS?

- [ ] для кожного стратегічного напрямку зафіксовано, від чого я відмовляюсь
- [ ] немає “безкоштовних” стратегій без ціни

👉 якщо ні → рівень невалідний

---

## 6. ЧИ Є RESOURCE REALISM?

- [ ] стратегічні напрямки підйомні в поточних або планованих ресурсах
- [ ] явно непідйомні напрямки спрощені, відкладені або відсіяні

👉 якщо ні → рівень невалідний

---

## 7. ЧИ Є FREEZE?

- [ ] результат зафіксовано
- [ ] після сесії не продовжується нескінченне перепроектування
- [ ] є готовність передати результат у довгострокову стратегію

👉 якщо ні → рівень невалідний

---

## 8. DONE CONDITION

Рівень вважається достатньо готовим якщо:

- [ ] стратегічно релевантні орієнтири покриті
- [ ] фактори реалізації визначені
- [ ] стратегічні напрямки реалізації визначені
- [ ] напрямки мають trade-offs
- [ ] результат достатній для інтеграції в довгострокову стратегію

---

## 9. УМОВНА ГОТОВНІСТЬ

Можна позначити як УМОВНО ГОТОВО якщо:

- [ ] основна логіка проектування вже зібрана
- [ ] залишилися другорядні уточнення
- [ ] є DEADLINE доопрацювання
- [ ] вже можливо інтегрувати в довгострокову стратегію без критичної шкоди

👉 у такому випадку:
УМОВНО ГОТОВО ДО: [Дата]
