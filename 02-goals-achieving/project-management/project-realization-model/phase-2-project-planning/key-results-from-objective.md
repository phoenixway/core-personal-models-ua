# 🧭 Методологія отримання Key Results з туманної цілі

## 0. ВАЛІДАЦІЯ ЦІЛІ

👉 Чи варто взагалі робити KR.

### Що це

Фільтр перед інвестицією життя.

Багато цілей красиві, але не твої.
KR на неправильну ціль = ідеально організована катастрофа.

---

### Питання

1. Якщо я проживу життя на 70%, чи це буде ок?
2. Від якої помилки ця ціль захищає?
3. Яку ціну доведеться платити?
4. Що гірше: не досягти її чи прожити без неї?
5. Чи витримає вона 10 років рутини?

---

### Вихід

👉 Одне з двох:

* ✔ Ціль прийнята
* ❌ Ціль відкинута

👉 Записати коротку формулу:

```
Objective:
[Дієслово] + [Об’єкт] + [Для чого]
```

Приклад:

```
Навчитись функціональній гармонії,
щоб імпровізувати для душі на піаніно.
```

---

## 1. КРИСТАЛІЗАЦІЯ РЕЗУЛЬТАТУ

👉 Що означає «досягнуто».

Без цього KR неможливі.

---

### Крок 1. Формула «Зараз → Потім»

```
Зараз: …
Після: …
```

Приклад для твоєї музики:

```
Зараз: гублюсь у ритмі при читанні нот
Після: читаю прості п’єси у 4/4 і 3/4 без помилок
```

---

### Крок 2. Питання «Що зміниться у світі»

Не в голові. У реальності.

* Що можна сфотографувати?
* Який документ з’явиться?
* Що стороння людина побачить?

Приклади:

* 10 записаних композицій
* Git-репозиторій з 5 сервісами
* 90% котів виписані без ускладнень

---

### Крок 3. Виміри реальності

Будь-який результат проявляється в площинах:

* кількість
* якість
* швидкість
* надійність
* вартість

Вибрати 2–3.

---

### Вихід

👉 Список конкретних ознак досягнення.

```
Ознаки результату:
1.
2.
3.
```

---

## 2. ГЕНЕРАЦІЯ МЕТРИК

👉 Як це виміряти.

---

### Крок 1. Список кандидатів

Написати 10–20 метрик.

Підказки:

* output (що створено)
* outcome (що змінилось)
* behavior (що роблю)
* ресурси
* помилки

Не фільтрувати.

---

### Крок 2. Фільтр GOOD METRIC

Метрика має бути:

* Observable – можна порахувати
* Controllable – ти впливаєш
* Meaningful – веде до цілі
* Timely – змінюється швидко
* Hard to fake – не накрутиш

---

### Крок 3. Вибір 3–5 метрик

Бо увага – дефіцит.

---

### Вихід

```
Метрики:
1.
2.
3.
```

---

## 3. ПРИЧИННА МОДЕЛЬ

👉 Від чого залежать метрики.

Без цього KR будуть випадковими.

---

### Крок

Для кожної метрики написати:

```
Метрика:
 ├ Фактор 1
 ├ Фактор 2
 └ Фактор 3
```

Приклад для притулку:

```
Смертність кошенят ↓
 ├ ранній діагноз сепсису
 ├ правильні інфузії
 └ навчання волонтерів
```

---

### Питання

* Які фактори під моїм контролем?
* Які мають найбільший вплив?
* Які дії лише створюють ілюзію роботи?

---

### Вихід

👉 3–7 факторів з найбільшим впливом.

---

## 4. ФОРМУВАННЯ KEY RESULTS

---

### Формула KR

```
До [дата]
метрика [назва]
зміниться з [A] до [B]
```

---

### Типи KR

#### 1. Outcome

Результат у світі.

```
Смертність ↓ з 30% до 10%
```

#### 2. Leading

Те, що ти робиш.

```
100% котів проходять чек-лист огляду
```

#### 3. Health

Щоб система не зламалась.

```
Середній сон ≥ 7 год
```

---

### Перевірка KR

Постав питання:

* Чи можна виконати без прогресу? → поганий KR
* Чи це просто задача? → переписати
* Чи видно результат без пояснень? → добре

---

### Вихід

👉 3–5 чітких KR.

---

## 5. PRE-MORTEM ПЕРЕВІРКА

👉 Чому ці KR проваляться.

Метод від психолога Gary Klein.

---

### Кроки

1. Уяви: через рік провал.
2. Напиши 10 причин.
3. Згрупуй.
4. Для кожної придумай метрику-запобіжник.

---

### Приклад

```
Причина: нові волонтери помиляються
→ KR: навчання 1× на місяць
```

---

### Вихід

👉 Додані Health KR або уточнені KR.

---

## ⚔️ Маленький приклад 

Ціль: функціональна гармонія.

```
Objective:
Навчитись функціональній гармонії
щоб імпровізувати для душі.

Ознаки:
– можу гармонізувати мелодію
– імпровізую в 5 тональностях
– записано 5 п’єс

KR:
1. До 3 міс гармонізувати 20 мелодій
2. До 2 міс знати 12 функцій у 5 тональностях
3. 30 днів підряд 20 хв вправ без пропуску
```
