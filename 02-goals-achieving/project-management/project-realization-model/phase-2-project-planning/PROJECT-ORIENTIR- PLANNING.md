# PROJECT / LIFE ORIENTIR PLANNING

## 0. Принципи
- Ресурсний ліміт: виділений, незмінний час/енергія.
- Чіткість → вимірюваність → неминучість виконання.
- Відкидання зайвого (80/20).
---
## 1. Why
Зараз ваш Objective (Блок 1) описує кінцевий стан ("Я маю..."). Це чудово, але іноді для довгострокової мотивації не вистачає відповіді на питання "Навіщо?".
Пропозиція: Перед визначенням Objective додати коротке формулювання Візії або фундаментальної причини.
Навіщо це? Objective — це що ви хочете. Візія — це чому ви цього хочете. Коли ви у "довгій грі" (горизонт "Рік"), саме "Навіщо?" допомагає не кинути справу, коли стикаєтесь із Barriers (Блок 6).
## 2. Objective (Орієнтир / Кінцевий стан)
**Форма:** “Я маю / Я вмію / Існує система…”  
**Вимоги:** конкретно, перевіряється, описує реальність, не процес.  
**Output:** один абзац + 1 фраза “я знаю, що це досягнуто, коли...”.
## 3. Key Results (Метрики досягнення)
**Вимірювані докази завершення Objective.**  
**Типи KR:**  
- Метрика → цифра  
- Факт / Документ  
- Компетенція  
- Артефакт / Система  

Після того, як ви визначили список KR, присвойте кожному з них пріоритет (наприклад, P1, P2, P3). 

Це змушує прийняти рішення: що є "1 ключовий прорив" (згаданий у Блоці 3), а що є другорядним. Це допомагає, коли ресурсний ліміт (час/енергія) раптово скорочується. Ви точно знаєте, що можна відкинути (P3), а що — ні (P1).
## 4. Ієрархія цілей (Декомпозиція у часі)

| Горизонт | Що тут визначається |
|----------|------------------------|
| Рік | 1–3 стратегічні Result (KR рівня року) |
| Квартал | Підцілі + пріоритетні **Ініціативи** (з Блоку 3) |
| Місяць | Орієнтири місяця + 1 ключовий прорив (з Ініціатив) |
| **Тиждень** | **Список Ключових Активностей:** <br/> 1. Конкретні результати/дії з **Ініціатив**. <br/> 2. Задачі з **Функцій**. |
| **День** | Пріоритетна активність + Мінімальний незламний крок. |

Після того, як ви визначили список, присвойте кожному з них пріоритет (наприклад, P1, P2, P3).

## 4. Functions / Systems (Системна діяльність)
- Що має працювати регулярно (щодня/щотижня), щоб KR стали неминучими.
- Це не завдання, а роль/процес.

**Формат:**  
`[Система/Функція] → [Випереджаючий показник (Lead)] → [Цільовий KR (Lag)]`
Приклади:
- Система: Контроль здоров'я
	- Lead-показник: Щоденне споживання <= 1900 ккал
	- Цільовий KR (Lag): Вага 75 кг
- Система: Вивчення Android Dev
	- Lead-показник: 60 хвилин фокусованої практики щодня
	- Цільовий KR (Lag): Компетенція "Вмію створювати multi-module Hilt-додаток"
- Система: Контент-маркетинг
	- Lead-показник: 2 оптимізовані статті опубліковано щотижня
	- Цільовий KR (Lag): 10k трафіку на місяць

## 5. Competencies & Resources
Що треба **мати / вміти / забезпечити**, щоб система працювала.

Категорії:
- Навички  
- Знання  
- Інструменти  
- Люди / підтримка  
- Стан (сон, концентрація, енергія)

## 6. Barriers & Risks
- Що може зупинити систему?  
- Як це мінімізується або обхідним шляхом вирішується?

---

## 7. Несуча конструкція (Carcass)
Система вважається **живою**, якщо одночасно існують:

| Блок | Перевірка, що він є |
|------|----------------------|
| Система цілей | Є Objective, KR, план тижня, план дня |
| Система енергії | Сон, фізика, харчування стабільні |
| Система уваги | Щоденний фокус → виконання → фіксація |
| Зворотний зв'язок | Щотижневий review+корекція |
| Середовище | Мінімум шуму, потрібні інструменти, підтримуючі люди |

---

## 8. Review Loop (Цикл зворотного зв'язку)
*(Без цього усе руйнується)*

| Горизонт | Фокус | Ключове питання / Дія |
|:---|:---|:---|
| **Щодня** | **Трекінг Lead-показників (Дій)** | "Чи виконав я те, що на 100% контролюю?" <br/> (Напр., "Чи з'їв я <= 1900 ккал?", "Чи практикувався я 60 хв?"). <br/> *Це дає відчуття контролю та щоденні "перемоги".* |
| **Щотижня** | **Аналіз зв'язку Lead → Lag (Результат)** | "Чи мої *дії* (Leads) ефективно рухають мої *метрики* (Lags)?" <br/> (Напр., "Я тримав 1900 ккал (Lead), але вага (Lag) стоїть. Чи варто змінити гіпотезу/Lead на 1800 ккал?"). <br/> *Це аналіз ефективності та корекція гіпотези.* |
| **Щомісяця** | **Стратегічна звірка (Напрямок)** | "Чи рухаються мої *метрики* (KRs / Lags) до *Орієнтиру* (Objective)?" <br/> *Це оновлення фокусу та пріоритезація Ініціатив на наступний місяць.* |
