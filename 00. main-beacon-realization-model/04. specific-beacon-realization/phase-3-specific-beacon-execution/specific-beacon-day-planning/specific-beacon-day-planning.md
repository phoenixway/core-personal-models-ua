## Вимоги до алгоритму
**Key Result mbr = у будь-який момент часу я знаю (або маю записаним), що саме зараз треба робити для кожного орієнтира, щоб прогрес був гарантований достатнім.**

Отже, треба **алгоритм**, який:
1. перетворює кожен життєвий орієнтир → у конкретний стан/вектор руху,
2. постійно оновлює відповідь на питання «що робити зараз?»,
3. гарантує, що ця відповідь не випадкова, а випливає зі структури.

Нижче — сформований **робочий алгоритм**, який можна реалізувати технічно
## ✅ **Алгоритм “What Now?” для кожного життєвого орієнтира**

### **Крок 0. Статичне — один раз робиться для кожного орієнтира**

Для кожного життєвого орієнтира створюється його “паспорт”:

| Поле                                  | Що там є                                                                                  |
| ------------------------------------- | ----------------------------------------------------------------------------------------- |
| **Objective**                         | кінцевий стан, як виглядає реалізований орієнтир                                          |
| **KR (Key Markers)**                  | 1–3 маркери, що підтверджують рух/реальність (не ідеал, а мінімальний робочий стан)       |
| **Функція**                           | постійна роль або дія, без якої прогрес зупиниться (наприклад: "щодня 1 год розбудови X") |
| **Мінімальний крок (“action floor”)** | дія, яка повинна виконуватися навіть у найгіршому стані (напр. 10 хв роботи без перерв)   |
| **Активний проєкт**                   | конкретний короткий фокус, через який зараз реалізується орієнтир                         |

---

### **Крок 1. Динамічне — як отримувати відповідь “що робити зараз?”**

Алгоритм кожного разу, коли ти ставиш питання “що я маю робити зараз?”:

```
1. Обрати орієнтир → наприклад “здоров’я”, “розвиток”, “свобода”, “створення системи X”.

2. Перевірити активний стан:
   – Чи є поточний проєкт для цього орієнтира?
     • Якщо НІ → вибрати один (найближчий до KR).
     • Якщо ТАК → перейти до п.3.

3. Задати 3 питання:
   a) Що є наступним кроком активного проєкту? (конкретна дія)
   b) Чи це відповідає мінімальному стандарту (action floor)?
   c) Чи дає це рух хоча б по одному Key Result?

4. Якщо відповідь «так» → це і є “що робити зараз”.

5. Якщо відповідь «ні» → обрати новий найближчий наступний мікрокрок або скорегувати проєкт.

6. Зафіксувати відповідь (1 рядок) у “Current Action Sheet”.
```
### **Крок 2. Як гарантувати, що відповідь завжди актуальна?**

Створюється **цикл оновлення**:

|Частота|Що оновлюється|
|---|---|
|Щодня|на початку дня → “Для кожного орієнтира → який активний проєкт → який наступний крок?”. Це дає список “що роблю сьогодні по кожному”.|
|Щотижня|рев’ю: чи активні проєкти просувають KRs; якщо ні → замінити.|
|У довільний момент|якщо запит “що робити зараз?” → просто дивлюсь в “Current Action Sheet”.|

### **Структура “Current Action Sheet” (завжди актуальна)**

```
| Орієнтир | Активний проєкт | Наступна дія | Часовий слот / мінімум |
|----------|------------------|--------------|--------------------------|
| Здоров’я | Сила/сон         | 20 хв силових + запис сну  | до 22:00 лягти |
| Професія | Проєкт X         | реалізувати модуль Y       | 1 глибока сесія |
| Фінанси  | Сервіс Z         | завершити прототип         | 1 Pomodoro |
| Воля/розум | Концентрація   | 10 хв медитації            | одразу після кави |
```

Це — **моментальна відповідь на твоє KR: “що я зараз маю робити?”**

### ✅ **Коротка формула**
```
Орієнтир → Objective → Key Results → Активний Проєкт → Наступна дія → Записано.
```

**Якщо кожен орієнтир має такий “живий ланцюг” → ціль досягнута.**
