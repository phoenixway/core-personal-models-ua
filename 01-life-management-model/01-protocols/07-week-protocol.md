# 📅 ПРОТОКОЛ — ПЛАНУВАННЯ ТИЖНЯ

---

## ВХІД

* місячна програма
* capacity (поточний стан)

---

# ⏱ КРОК 0 — TIMEBOX

```text id="6bxt8k"
TIMEBOX: 20–40 хв

❗ без timebox протокол не запускається  
❗ після завершення → результат фіксується (freeze)
```

---

# 🔍 КРОК 1 — ВИБІР ЦІЛЕЙ

👉 взяти з місячної програми

```text id="hpk7c3"
обрати 1–3 цілі тижня
```

---

## RULE

```text id="zt6z5l"
жодних нових напрямків
```

---

## DONE CONDITIONS

* цілі прямо відповідають місяцю
* не є задачами
* перевіряються

---

# 🧠 КРОК 2 — ДЕКОМПОЗИЦІЯ

Для кожної цілі:

```text id="c7pt6k"
напрямки роботи (2–4)
```

---

## RULE

```text id="o4d9ny"
це не задачі, а типи активності
```

---

## ПРИКЛАД

* дослідження
* реалізація
* тестування
* стабілізація

---

# ⚙️ КРОК 3 — ACTION READINESS

👉 перевірити:

```text id="4z9hvt"
чи можна з цього отримати конкретні дії?
```

---

## BOUNCE

```text id="1k3mza"
якщо ні → напрямок занадто розмитий
```

---

# ⚖️ КРОК 4 — LOAD CHECK

```text id="9b6txv"
перевірити:

- чи реалістично це виконати за тиждень?
- чи не перевантажено?
```

---

## RULE

```text id="2y5b4n"
якщо сумнів → скоротити
```

---

## BOUNCE

```text id="n3j7pw"
якщо:

- більше 3 цілей
- забагато напрямків

→ скоротити
```

---

# 🚫 КРОК 5 — EXCLUSIONS

```text id="z7q9lf"
що НЕ роблю цього тижня?
```

---

## RULE

```text id="4t2m8v"
тиждень без exclusions = розфокус
```

---

# 🔗 КРОК 6 — ТРАНСЛЯЦІЯ

👉 перевірити:

```text id="6y5r2n"
тиждень → рухає місяць?
```

---

## RULE

```text id="8p4n1k"
якщо ні → це шум
```

---

# 🧊 КРОК 7 — FREEZE

```text id="w6r2xk"
план тижня фіксується

- не оптимізується
- використовується як є
```

---

## ВИНЯТОК

```text id="3z7v1c"
перегляд тільки якщо:

- зміна реальності
- системний збій
```

---

# 📦 ВИХІД

```text id="9k2x7q"
- 1–3 цілі тижня
- напрямки роботи
- exclusions
```

---

# ✅ ВАЛІДНІСТЬ

```text id="5x1v8d"
- ≤ 3 цілі
- всі пов’язані з місячною програмою
- є exclusions
- є action readiness
- є capacity fit
- сформовано в межах timebox
```

---

# ⚠️ ПРАВИЛА

1. більше 3 цілей = перевантаження
2. нові напрямки = помилка
3. без exclusions = розфокус
4. якщо не можна діяти → план неготовий
5. не оптимізувати після freeze
6. тиждень — це execution layer

---

# ⚡ КОРОТКО

1. вибрати ≤3
2. розкласти на напрямки
3. перевірити action readiness
4. перевірити load
5. exclusions
6. freeze
