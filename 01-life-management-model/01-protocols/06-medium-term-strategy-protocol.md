# 🚀 ПРОТОКОЛ — ФОРМУВАННЯ МІСЯЧНОЇ ПРОГРАМИ

---

## Inbox

# ⏱ КРОК 0 — TIMEBOX

```text
TIMEBOX: 30–90 хв

❗ без timebox протокол не запускається
❗ після завершення → результат фіксується (freeze)
```

---

# 📊 КРОК 1 — CAPACITY PROFILE

👉 оцінити РЕАЛЬНУ пропускну здатність

```text
CAPACITY PROFILE:

- deep work slots: __
- admin/support load: __
- maintenance baseline: __
- volatility reserve: __%
- recovery reserve: __%

- max concurrent missions: ≤ 3
```

---

## RULE

```text
maintenance враховується ДО планування прогресу
```

---

## VALIDATION

```text
missions_load + maintenance + reserve ≤ capacity
```

❗ якщо не сходиться → скоротити обсяг

---

# 🔍 КРОК 2 — ВИБІР НАПРЯМКІВ

- взяти зі стратегії
- максимум 2–3

---

## DONE CONDITIONS

- не є задачами
- відповідають ключовим факторам
- не конфліктують між собою
    
---

# 🎯 КРОК 3 — ФОРМУВАННЯ ЦІЛЕЙ

Для кожного напрямку:

```text
ЦІЛЬ:

- чіткий результат
- перевіряється
- досяжна в межах місяця
```

---

## DONE CONDITIONS

- 1–3 цілі
- кожна має результат (не “працювати над…”)ґ
- кожна вкладається в capacity
    

---

# 🚫 КРОК 4 — EXCLUSIONS (ОБОВ’ЯЗКОВО)

👉 що НЕ роблю цього місяця

```text
EXCLUSIONS:

- напрямки
- задачі
- ініціативи
```

---

## RULE

```text
кожна ціль витісняє іншу можливу ціль
```

---

# ⚖️ КРОК 5 — LOAD CHECK

```text
перевірити:

- кількість цілей ≤ 3
- немає перевантаження
- є резерв
```

---

## BOUNCE

```text
якщо:

- перевантаження
- немає резерву
- нереалістичний обсяг

→ повернутись до скорочення
```

---

# 🔗 КРОК 6 — ТРАНСЛЯЦІЯ

👉 перевірити:

```text
стратегія → місяць
```

- чи рухає це стратегічні напрямки?
- чи не є це випадковими задачами?

---

# 🧊 КРОК 7 — FREEZE

```text
результат фіксується

- не покращується
- не переробляється
- використовується як є
```

---

## ВИНЯТОК

```text
дозволено перегляд тільки якщо:

- виявлена блокуюча помилка
- або суттєва зміна реальності
```

---

# 📦 ВИХІД

```text
- 1–3 цілі
- capacity profile
- список exclusions
```

---

# ✅ ВАЛІДНІСТЬ

Місячна програма валідна якщо:

```text
- ≤ 3 цілі
- є exclusions
- є capacity fit
- є резерв
- відповідає стратегії
- створена в межах timebox
```

---

# ⚠️ ПРАВИЛА

1. більше 3 цілей = помилка
2. без exclusions = помилка
3. без capacity = ілюзія
4. перевантаження = системний дефект
5. не оптимізувати після freeze
6. execution починається одразу після

---

# 🔁 ПІСЛЯ

👉 перейти в:

```text
week planning → day → execution
```
