# 🧭 ПРОТОКОЛ — СТРАТЕГІЧНЕ ПРОЕКТУВАННЯ РЕАЛІЗАЦІЇ ГОЛОВНИХ ОРІЄНТИРІВ

👉 Мета:
для головних орієнтирів, які потребують стратегічної або змішаної реалізації,
визначити фактори реалізації і стратегічні напрямки реалізації

👉 Вхід:
- [[00-main-beacons]]
- [[01-main-beacon-realization-models]]
- сигнали з [[05-control-mode]] (якщо є)

👉 Вихід:
- стратегічне проектування реалізації головних орієнтирів
- для кожного релевантного орієнтиру:
  - фактори реалізації
  - 2–3 стратегічні напрямки реалізації
  - trade-offs
  - логіка впливу

---

## ⏱ КРОК 0 — TIMEBOX

```text
TIMEBOX: 45–90 хв

❗ без timebox протокол не запускається
❗ після завершення → результат фіксується (freeze)
````

---

## КРОК 1 — ВИБІР ОРІЄНТИРІВ ДЛЯ СТРАТЕГІЧНОГО ПРОЕКТУВАННЯ

Взяти тільки ті головні орієнтири, для яких у моделі реалізації визначено:

- стратегічний тип
    
- змішаний тип
    

---

### RULE

```text
проектні або чисто підтримуючі орієнтири тут не проектуються стратегічно
```

---

## КРОК 2 — ВИЯВЛЕННЯ ФАКТОРІВ РЕАЛІЗАЦІЇ

Для кожного релевантного орієнтиру відповісти:

```text
- що обмежує реалізацію?
- що дає найбільший ефект?
- що може зламати реалізацію?
- що незрозуміло?
```

👉 виписати всі відповіді як кандидати у фактори реалізації

---

### RULE

```text
фактор реалізації = умова, яка суттєво впливає на реалізацію орієнтиру
```

---

## КРОК 3 — ФОРМАЛІЗАЦІЯ ФАКТОРІВ

Кожен фактор записується у причинно-наслідковій формі:

```text
[умова] → [вплив на реалізацію орієнтиру]
```

---

### ПРИКЛАДИ

- нестабільний дохід → неможливо регулярно інвестувати в розвиток
    
- низька енергія → не відбувається системна творча реалізація
    
- відсутність публічного каналу → вклад не масштабується
    
- невизначеність щодо ринку → стратегія заробітку не збирається
    

---

### RULE

```text
без причинно-наслідкового зв’язку це не фактор
```

---

## КРОК 4 — ФІЛЬТР ФАКТОРІВ

Відкинути:

- дії
    
- задачі
    
- бажання
    
- красиві абстракції без механіки
    
- шум, що не впливає на реалізацію орієнтиру
    

---

### РОЗШИРЕНИЙ ФІЛЬТР

```text
якщо:
- це можна зробити сьогодні → це дія, не фактор
- немає механіки впливу → не фактор
- не змінює реалізацію орієнтиру → шум
```

---

### BOUNCE

```text
- дія → вниз (execution / planning)
- розмите → уточнити
- не впливає → видалити
```

---

## КРОК 5 — ТИПІЗАЦІЯ ФАКТОРІВ

Для кожного фактора визначити TYPE:

- constraint
    
- leverage
    
- risk
    
- uncertainty
    

---

### RULE

```text
тип фактора визначає спосіб стратегічної реакції
```

---

## КРОК 6 — ПРИВ’ЯЗКА ФАКТОРІВ

Для кожного фактора зафіксувати:

```text
цей фактор впливає на який головний орієнтир?
```

---

### RULE

```text
фактор без прив’язки до конкретного орієнтиру → видалити
```

---

## КРОК 7 — ОЦІНКА ФАКТОРІВ

Для кожного фактора оцінити:

- важливість (1–5)
    
- вплив (1–5)
    
- терміновість (1–5)
    
- керованість (1–5)
    

---

### OPTIONAL

```text
SCORE = impact + urgency + (importance × 2)
```

---

## КРОК 8 — ВИБІР АКТИВНИХ ФАКТОРІВ

Для кожного головного орієнтиру:

```text
вибрати ≤ 5 активних факторів реалізації
```

---

### RULE

```text
активний фактор витісняє інший
```

---

### BOUNCE

```text
якщо >5 → скоротити
```

---

## КРОК 9 — ТРАНСФОРМАЦІЯ ФАКТОРІВ У СТРАТЕГІЧНІ НАПРЯМКИ

Для кожного активного фактора запитати:

```text
як змінити ситуацію стратегічно?
```

---

### RULE ПО ТИПУ

```text
constraint → прибрати / обійти
leverage → посилити
risk → зменшити / контролювати
uncertainty → прояснити
```

---

👉 результат: чорнові стратегічні напрямки реалізації

---

## КРОК 10 — ФОРМАЛІЗАЦІЯ СТРАТЕГІЧНИХ НАПРЯМКІВ

Кожен напрямок записати як:

```text
СТРАТЕГІЧНИЙ НАПРЯМОК РЕАЛІЗАЦІЇ:

- що саме змінюємо
- як це впливає на реалізацію орієнтиру
```

---

### RULE

```text
це не задача, а стратегічний спосіб руху
```

---

### ПРИКЛАДИ

- стабілізація доходу → з’являється база для фінансової реалізації орієнтиру
    
- розвиток публічного контуру творчості → вклад починає лишати слід
    
- захист часу deep work → виникає простір для великої реалізації
    
- прояснення робочої ніші → зменшується стратегічна невизначеність
    

---

## КРОК 11 — ФІЛЬТР СТРАТЕГІЧНИХ НАПРЯМКІВ

Відкинути:

- задачі
    
- конкретні кроки
    
- абстракції без механіки
    
- напрямки без реального впливу на орієнтир
    

---

### BOUNCE

```text
- дія → вниз
- розмите → уточнити
- не впливає → видалити
```

---

## КРОК 12 — ПРИВ’ЯЗКА НАПРЯМКІВ

Для кожного напрямку зафіксувати:

- який фактор він закриває
    
- який головний орієнтир він обслуговує
    

---

### RULE

```text
напрямок без фактору або без орієнтиру → шум
```

---

## КРОК 13 — TRADE-OFF

Для кожного стратегічного напрямку запитати:

```text
від чого я відмовляюсь заради цього?
```

---

### RULE

```text
якщо немає відмови → це ще не стратегія
```

---

## КРОК 14 — RESOURCE CHECK

Перевірити для кожного напрямку:

```text
чи підйомно це в поточних або планованих ресурсах?
```

---

### BOUNCE

```text
якщо ні → спростити / відкласти / знизити пріоритет
```

---

## КРОК 15 — ОБ’ЄДНАННЯ І ВИБІР

Для кожного головного орієнтиру:

- об’єднати близькі напрямки
    
- прибрати дублікати
    
- залишити 2–3 стратегічні напрямки реалізації
    

---

### RULE

```text
кожен стратегічний напрямок витісняє інший
```

---

## КРОК 16 — FREEZE

Результат фіксується:

- не оптимізується далі в цій сесії
    
- використовується як є
    
- інтегрується в довгострокову стратегію
    

---

### ВИНЯТОК

Перегляд тільки якщо:

- з’явились нові дані
    
- в control знайдений системний дефект
    
- змінилась модель реалізації орієнтиру
    

---

# 📦 ВИХІД

Для кожного стратегічно релевантного головного орієнтиру:

- список факторів реалізації
    
- ≤ 5 активних факторів
    
- 2–3 стратегічні напрямки реалізації
    

Кожен напрямок має:

- напрям
    
- фактор
    
- орієнтир
    
- trade-off
    
- логіку впливу
    

---

# ✅ ВАЛІДНІСТЬ

Стратегічне проектування валідне якщо:

- воно зроблене тільки для орієнтирів, яким це потрібно
    
- фактори не є діями
    
- фактори мають причинно-наслідкову форму
    
- фактори прив’язані до конкретних орієнтирів
    
- для кожного релевантного орієнтиру є ≤ 5 активних факторів
    
- стратегічні напрямки не є задачами
    
- стратегічні напрямки пов’язані з факторами
    
- стратегічні напрямки мають trade-offs
    
- усе сформовано в межах timebox
    

---

# ⚠️ ПРАВИЛА

1. дія ≠ фактор
    
2. фактор без causal form = не фактор
    
3. фактор без орієнтиру = шум
    
4. стратегічний напрямок без фактору = шум
    
5. без trade-off = не стратегія
    
6. більше 3 стратегічних напрямків = розпорошення
    
7. не оптимізувати після freeze
    
8. стратегічне проектування не підміняє довгострокову стратегію, а готує вхід для неї
    

---

# ⚡ КОРОТКО

1. вибрати стратегічно релевантні орієнтири
    
2. знайти фактори реалізації
    
3. формалізувати і відфільтрувати
    
4. типізувати й оцінити
    
5. вибрати активні фактори
    
6. трансформувати їх у стратегічні напрямки
    
7. додати trade-offs
    
8. вибрати 2–3 напрямки на орієнтир
    
9. freeze
