# ⚙️ ПРОТОКОЛ — УПРАВЛІННЯ ДНЕМ

👉 Мета:
перетворити тижневий фокус і action units у реальний день виконання

👉 Вхід:
- режим дня
- action units
- ситуативні задачі
- поточний стан / ресурс

👉 Вихід:
- план дня
- виконані задачі
- виконані action units
- реальний прогрес

---

## 📥 ВХІД

- day-start
- новий контекст:
  - актуальний рівень
  - активні напрямки
  - цілі періоду
- LM action units із `plan_today_life_management`
- ситуативні задачі дня

---

# ⏱ КРОК 0 — TIMEBOX ПЛАНУВАННЯ

TIMEBOX: 5–10 хв

❗ без timebox планування дня не запускається  
❗ після завершення → план фіксується

---

# 🧠 КРОК 1 — ПЕРЕВІР ГОТОВНІСТЬ ДО ДНЯ

Запитай:

- чи є ясність?
- чи є хоча б мінімальний ресурс?
- чи не активний EMERGENCY / CRASH?

👉 якщо ні:
- спростити день
- зменшити навантаження
- перейти до мінімального варіанту

---

# 📅 КРОК 2 — ВИБІР ФОКУСУ ДНЯ

Обрати 1–2 ключові фокуси дня

Фокус дня = те, що сьогодні реально рухається

---

## RULE

день не може мати > 2 фокусів

❗ > 2 фокусів = розпад дня

---

# 🔧 КРОК 3 — ACTION LAYER

👉 день будується через action units

не:
тиждень → день

а:
тиждень → action units → день

---

## ВИБРАТИ

взяти на сьогодні:

- 1–3 action units максимум

---

## RULE

action unit = мінімальна завершена одиниця руху

---

## ПРИКЛАДИ

- завершити модуль X
- провести 2 години deep work над Y
- закрити задачу Z
- зробити review дефекту D-001

---

# 🧩 КРОК 4 — ДЕКОМПОЗИЦІЯ В ЗАДАЧІ

Для кожного action unit:

- розбити на 2–5 конкретних задач

---

## RULE

задача = можна почати одразу

---

## BOUNCE

якщо не ясно як почати → задача неправильна  
якщо задача занадто велика → розбити  
якщо задача розмита → переписати

---

# 📌 КРОК 5 — ДОДАТИ СИТУАТИВНІ ЗАДАЧІ

Додати:

- роботу
- обов’язки
- термінові речі

---

## RULE

ситуативні задачі не витісняють action units  
вони вписуються поруч, а не замість

---

# ⚖️ КРОК 6 — LOAD CHECK

Перевірити:

- чи реально це виконати за день?
- чи відповідає це моєму стану?
- чи не перевантажений день?

---

## HARD LIMIT

- 1–3 action units
- не більше 6–8 задач
- не більше 2 фокусів

---

## RULE

якщо є сумнів → скоротити  
якщо перевантаження → прибрати зайве  
якщо LOW state → залишити 1 action unit

---

# 🧊 КРОК 7 — FREEZE

План дня фіксується

- не змінюється хаотично
- виконується як є
- thinking завершується
- далі тільки execution

---

## ВИНЯТОК

дозволено змінити план тільки якщо:

- форс-мажор
- радикальна зміна реальності
- з’явився блокуючий дефект
- активувався EMERGENCY / RECOVERY

---

# 🚀 КРОК 8 — EXECUTION

👉 режим:
EXECUTION

---

## EXECUTION LOOP

1. взяти action unit
2. взяти наступну задачу з нього
3. виконати
4. зафіксувати результат
5. перейти до наступної задачі
6. завершити action unit
7. перейти до наступного

---

## RULE

EXECUTION ≠ THINKING

❗ під час виконання:
- не переплановувати
- не оптимізувати
- не відкривати новий improve
- не міняти фокуси

---

# 🔁 КРОК 9 — ПЕРЕХІД В ПРОЦЕСІ ДНЯ

👉 після виконання:

- завершив задачу → наступна
- завершив action unit → наступний
- завершив усі units → день завершено

---

## BOUNCE (під час дня)

якщо:

- завис → спростити задачу
- немає руху → повернутись до мінімальної дії
- кілька задач підряд не йдуть → це сигнал у control
- action unit не рухається → позначити для review / defect

---

# ✅ КРИТЕРІЙ ВИКОНАНОГО ДНЯ

День вважається виконаним якщо:

- виконані ключові задачі
- виконано хоча б 1 action unit
- є реальний рух по цілях
- не було перевантаження
- не було змішування thinking / doing

---

# 📦 ВИХІД

- список виконаних задач
- список виконаних action units
- реальний прогрес
- сигнали для day-review / control

---

# 🧾 ЗАВЕРШЕННЯ ДНЯ

→ day-review

Зафіксувати:

- що виконано
- які action units завершені
- де був розрив
- що треба передати далі

---

# ⚠️ ПРАВИЛА

1. день = execution
2. > 2 фокусів = розвал
3. без action units = ілюзія роботи
4. якщо не можна почати → задача неправильна
5. не оптимізувати після freeze
6. thinking тільки в окремій сесії
7. action units важливіші за другорядні ситуативні задачі
8. перевантаження дня = системна помилка, а не “слабкість”

---

# ⚡ КОРОТКО

1. перевір стан
2. обери 1–2 фокуси
3. візьми 1–3 action units
4. розбий на задачі
5. додай ситуативні
6. load check
7. freeze
8. execute

---

# 🧠 ФІНАЛ

День — це не список всього можливого  
і не “що встигну”

День — це:

👉 мінімальний набір виконуваних дій, які гарантовано рухають реальність

---

## Формула

орієнтири → фактори → стратегія → місяць → тиждень → action units → день → дія

👉 день = місце, де вся система реально штовхає світ
