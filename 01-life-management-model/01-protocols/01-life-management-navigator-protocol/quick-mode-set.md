# Швидке визначення режиму

- немає ясності → **IMPROVE** → Йди до [[04-improve-mode]] (Валідація рівнів)
- є ясність → **EXECUTION** → Йди до [[01-execution-mode]] (Виконання)
- щось не працює або прийшов час обов'язкового огляду→ **CONTROL** → Запусти [[05-control-mode]]]
- випав з контексту → **RECOVERY** → Відкрий Snapshot, не ламай систему [[02-recovery-mode]]
- перевантаження/криза → **EMERGENCY** → Скинь усі плани до 1 Action Unit на день [[03-emergency-mode]]
## ПРАВИЛА

- один режим одночасно  
- вищий режим перекриває нижчий  
- EXECUTION тільки якщо система стабільна  

### CONTROL (ОБОВ’ЯЗКОВО)
👉 запускається:
- раз на тиждень
- або при відчутті проблем
### SYSTEM CLOCK

- **Щодня** → Execution (Дія) + Micro-Improve (1 Timebox)
- **Щотижня** → Control + План тижня
- **Щомісяця** → Програма + Стратегічний Review
- **Періодично** → Перегляд Орієнтирів та Факторів
