# 🛠 IMPROVE MODE

👉 Мета:
покращити один рівень системи, отримати конкретний артефакт і згенерувати дії для виконання

👉 Результат:
- оновлений / новий артефакт рівня
- 1–3 ACTION UNITS
- передача в EXECUTION

---

## КОЛИ АКТИВУВАТИ

- є невалідний рівень
- немає ясності, що робити далі
- треба побудувати або уточнити рівень
- control показав, що потрібне системне покращення

❗ IMPROVE не є постійним режимом  
❗ це коротка сесія з обмеженим часом

---

## КРОК 1 — ВИЗНАЧ РІВЕНЬ

→ [[check-levels]]

👉 АКТУАЛЬНИЙ РІВЕНЬ = перший невалідний рівень  
або рівень, який свідомо обраний для покращення

❗ працюю тільки з одним рівнем

---

## КРОК 2 — ФІКСАЦІЯ СЕСІЇ

Перед запуском протоколу жорстко зафіксуй:

- 📌 РІВЕНЬ: [[Назва рівня]]
- ⏱ TIMEBOX: [ХХ хвилин]
- 🎯 ОЧІКУВАНИЙ РЕЗУЛЬТАТ: [Конкретний артефакт]

Приклад:

- РІВЕНЬ: [[03-long-term-strategy]]
- TIMEBOX: 90 хв
- ОЧІКУВАНИЙ РЕЗУЛЬТАТ: оновлена стратегія з 2 ставками і списком відмов

❗ без цієї фіксації управлінська робота не починається

---

## КРОК 3 — ЗАПУСК ПРОТОКОЛУ

👉 Відкрий протокол відповідного рівня і працюй тільки по ньому

- [[02-main-beacons-protocol]]
- [[03-main-beacon-realization-models-protocol]]
- [[04-main-beacon-strategic-projecting-protocol]]
- [[05-long-term-strategy-protocol]]
- [[06-medium-term-strategy-protocol]]
- [[07-week-protocol]]
- [[08-day-level-protocol]]

RULE:

- не змішую рівні
- не відкриваю інші протоколи
- не “допилюю все одразу”

---

## КРОК 4 — ПРАЦЯ В МЕЖАХ TIMEBOX

👉 Працюю тільки до завершення timebox

Під час сесії:

- формую або оновлюю артефакт
- не женуся за ідеальністю
- не продовжую “ще трохи”

❗ time is up = stop

---

## КРОК 5 — ФІКСАЦІЯ РЕЗУЛЬТАТУ

Після завершення timebox:

- зафіксувати артефакт рівня
- позначити, що саме стало ясним
- зафіксувати, що ще неготове, якщо рівень лише умовно валідний

Якщо рівень не ідеальний, але достатній для руху вниз:

- позначити його як УМОВНО ГОТОВИЙ
- поставити DEADLINE доопрацювання

---

## КРОК 6 — ПЕРЕХІД ДО ДІЇ

На основі нового / оновленого артефакту:

👉 згенеруй 1–3 ACTION UNITS

Для кожного:

- ЩО:
- РЕЗУЛЬТАТ:
- TIMEBOX:
- КУДИ ПЕРЕДАЄТЬСЯ: день / тиждень

Приклад:

1. ЩО: звузити стратегічний фокус до 2 ставок  
   РЕЗУЛЬТАТ: оновлений стратегічний артефакт  
   TIMEBOX: 30 хв  
   КУДИ ПЕРЕДАЄТЬСЯ: тиждень

2. ЩО: виписати 3 action units для реалізації нової стратегії  
   РЕЗУЛЬТАТ: список units  
   TIMEBOX: 20 хв  
   КУДИ ПЕРЕДАЄТЬСЯ: день

❗ якщо немає action units → протокол не завершений

---

## КРОК 7 — ПЕРЕДАЧА В EXECUTION

👉 Після генерації action units:

- перенеси їх у план дня / тижня
- заверш сесію improve
- перейди в EXECUTION

RULE:

- артефакт використовується навіть якщо неідеальний
- перегляд тільки в наступному циклі
- заборонено продовжувати improve без переходу до execution

---

## ВИНЯТОК

Дозволено ще 1 короткий timebox тільки якщо:

- виявлена блокуюча суперечність
- неможливо перейти вниз
- без цього артефакт не дає жодної дії

❗ не більше одного додаткового короткого timebox

---

## КОЛИ IMPROVE ВВАЖАЄТЬСЯ ЗАВЕРШЕНИМ

- обрано один рівень
- відпрацьовано в межах timebox
- отримано артефакт
- згенеровано 1–3 action units
- units передані в EXECUTION

❗ якщо немає дій на виході, improve не завершений

---

## ПРАВИЛА

1. IMPROVE = думаю, а не виконую  
2. Один improve = один рівень  
3. Без timebox improve заборонений  
4. Неідеальний артефакт, що дозволяє рухатись, = достатній  
5. Після improve обов’язковий перехід у execution  
6. Якщо проблема повторюється, а не вирішується → не “думати більше”, а йти в [[05-control-mode]]

---

## КОРОТКО

1. check-levels  
2. вибрати один рівень  
3. зафіксувати timebox і результат  
4. пройти протокол рівня  
5. зафіксувати артефакт  
6. створити 1–3 action units  
7. перейти в EXECUTION
